Solarcoin Claiming API API Reference

Welcome to the SolarCoin community! This API is designed for affiliates to file their claims programmatically. If you haven’t already, you’ll need to register at the SolarCoin Foundation website receive an access token prior to filing claims.

If you have questions, please direct them to affiliates@solarcoin.org or to the Slack channel.

Terms of Service: https://solarcoin.org/
Request Content-Types: application/json
Response Content-Types: application/json
Schemes: https
Version: 1.0.0

Authentication

basicAuth

All interaction with the API must be via using https (TLS). Our API uses HTTP Basic Auth, you should use your unique username in the login field and your issued token as the password.

Remember to keep these secure! Anyone possessing these credentials will be able to pose as you when interacting with the SolarCoin Foundation. Do not share the login, do not check it into a public Git repository, and do not embed it in client-side code.

Claim Registration

This endpoint is used for initial registration of a claim.

The SolarCoin Foundation supports two types of claims for granting: actual and estimated. Actual claims are based on output from inverter or meter readings, estimated claims are based on the nameplate capacity of the facility.

If the kwhAtClaimingDate property is present in a given project, the claim will be treated as actual, and an initial grant will be issued based on that property. The SCF will expect subsequent inverter or meter readings at the /claim/actual endpoint (described below) for future grants. The response object will contain a project ID - it is important to retain this, as it will be required to file the future inverter or meter readings for granting.

If the kwhAtClaimingDate property is missing, the claim will be treated as estimated, and will be granted using our standard estimation of 15% of the installation's nameplate capacity. No further submissions will be necessary.

All grants include prior generation, beginning with the installDate or 2010-01-01, whichever is later. Both actual and estimated grants will be issued every six months.

Schema Definitions

Project

Details regarding a solar generation facility

address: string
city: string
state: string
zipCode: string
country: string
The address of the solar facility. Country should be a two-character country code.
nameplate: number
The nameplate capacity of the solar facility in kWh
kwhAtClaimingDate: number
The production reading (in kWh) at the time of claiming. If present, this will register the facility as receiving grants based on actual production, and will require subsequent readings to be filed via the /claim/actual endpoint.
installDate: string
The installation date of the solar facility in DD-MM-YYYY format
walletAddress: string
The SolarCoin wallet address to which the grant should be paid
documentation: string
A public link to documentation proving energy generation
deviceID: string
An optional device ID. Can be the empty string.
legalOwner: string
The name of the legal owner of the solar facility
claimantRelationship: string
The relationship between the claimant and the the solar facility. Can be one of:
  • "owner"
  • "residential_resident"
  • "manager
  • "custodian"
ofacCompliant: boolean
Whether the Claimant has passed an OFAC check by the Affiliate. Can be one of:
  • "true"
  • "false"
Example
{
  "address": "string",
  "city": "string",
  "state": "string",

  "country": "string",

  "walletAddress": "string",
  "documentation": "string",
  "deviceID": "string",
  "legalOwner": "string",
  "claimantRelationship": "string",
  "ofacCompliant": "boolean"
}
